Act as an entity extraction engine.

Extract the following attributes from the given conversation:
- first_name
- last_name
- phone_number
- email
- budget (integer, INR only)
- current_location
- preferred_location
- profession
- visit_date (YYYY-MM-DD or null)
- buying_timeline_weeks (integer)

--------------------------------------------------
EXTRACTION RULES
--------------------------------------------------

GENERAL
- Extract values ONLY if they are explicitly mentioned in the conversation
- Do NOT infer, assume, or guess any value
- If a value is missing, unclear, or ambiguous, return null
- Respond with ONLY valid JSON
- Do NOT include explanations, comments, or extra text

NAME
- first_name and last_name must be extracted separately
- Capture only if explicitly stated
- Do NOT infer names from email
- If only one name is mentioned, treat it as first_name and set last_name to null

PHONE NUMBER
- Capture only if explicitly provided
- Otherwise return null

EMAIL
- Capture only if explicitly provided
- Otherwise return null

BUDGET
- Normalize to INR (numbers only)
- Use ONLY explicit budget mentions (ignore property prices or quoted values)
- Supported units:
  - Crore (1 crore = 10,000,000 INR)
  - Lakh (1 lakh = 100,000 INR)
- If a range is given, use the LOWER bound
- Examples:
  - "1.5 crore" → 15000000
  - "80 lakhs" → 8000000
  - "3–4 crore" → 30000000
- If budget is not explicitly stated, return null

LOCATIONS
- current_location: where the person currently stays
- preferred_location: where the person wants to buy
- Capture only if explicitly mentioned
- Otherwise return null

PROFESSION
- Allowed values ONLY:
  - service
  - business
  - retired
- If profession is unclear, outside these values, or not provided, return null

VISIT DATE
- Capture only if explicitly mentioned
- Format must be YYYY-MM-DD
- Current year is 2026
- If the mentioned date is NOT in 2026, return null
- If unclear or missing, return null

BUYING TIMELINE
- Convert to weeks (numbers only)
- Use LOWER bound if a range is given
- Conversions:
  - Days → weeks (1 week = 7 days, floor value)
  - Months → weeks (1 month ≈ 4 weeks)
  - Years → weeks (1 year = 52 weeks)
- Special cases:
  - "Immediately" → 0
  - "Day after tomorrow" → 0
  - "Next month" → 4
  - "Next year" → 52
- If unclear or not provided, return null

--------------------------------------------------
OUTPUT FORMAT
--------------------------------------------------

Return ONLY valid JSON in this exact structure:

{
  "first_name": string | null,
  "last_name": string | null,
  "phone_number": string | null,
  "email": string | null,
  "budget": integer | null,
  "current_location": string | null,
  "preferred_location": string | null,
  "profession": string | null,
  "visit_date": string | null,
  "buying_timeline_weeks": integer | null
}

Conversation:
{conversation}
